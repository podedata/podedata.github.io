# Case Contábil/Demonstrativo de Resultado - Podedata

## Informações Básicas
- Nome do Projeto: Reestruturação de dados para análise de demonstração de resultado
- Indústria/Segmento: Indústria
- Data/Período: 2025
- Ferramentas Utilizadas: (Power BI, Pentaho PDI, Python, Fabric MS)

## Contexto e Desafio
1. Qual era a situação inicial do cliente?
-> O cliente tinha planilhas para análise dos dados mensais que a partir de vários relatórios gerados, ia consolidando os conteúdos/dados para gerar a visão estritamente necessária para o mês, mas longe do desejado.
2. Quais eram os principais problemas/dores?
-> Tempo, complexidade na lógica de consolidação, visual minimamente entregável, possibilidade de erro na consolidação que poderia custar dias de trabalho 
3. Qual era o impacto no negócio?
-> Alto impacto

## Solução Implementada
1. O que foi feito? (resumo executivo)
2. Principais entregas:
	- Ambiente performático de extração de dados necessários para a composição do fato a ser analisado
	- Redução de tempo para a entrega do objetivo
	- Performance, qualidade e integridade dos dados
	- Visual desenhado conforme o gosto e necessidade do cliente para a análise do contexto
3. Tecnologias/Metodologias:
	- Mapeamento de demanda, planejamento
	- Levantamento e mapeamento dos dados
	- ETL e Engenharia de Dados
	- Modelagem e desenvolvimento de DAX
	- Desenvolvimento da Visualização e Storytelling
	- Entrega ao Cliente e Governaça
4. Timeline do projeto:
	- 3 meses

## Resultados e Impacto
1. Resultados quantitativos:
	- Redução de aproximadamente 94% do tempo total
2. Resultados qualitativos:
	- Tomada de decisão rápida e confiável
	- Melhor compreensão do contexto
	- Cultura de Dados
	- Alinhamento estratégico
